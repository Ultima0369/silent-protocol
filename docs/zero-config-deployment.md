# Silent Protocol — 零配置部署指南

> **给完全不懂技术的普通用户看的。**

---

## 三步上手

### 第一步：下载

从 GitHub Releases 页面下载 `silent-protocol-windows.zip`。

或者直接克隆仓库：

```
git clone https://github.com/Ultima0369/silent-protocol.git
```

### 第二步：双击 setup.bat

```
📁 silent-protocol/
   ├── setup.bat     ← 双击这个
   ├── run.bat       ← 以后双击这个
   ├── src/
   └── ...
```

双击 `setup.bat`，会问你要两样东西：

1. **API 地址** — 默认就是 DeepSeek 的地址，直接按回车
2. **API Key** — 从你的 AI 服务商那儿拿到的那串密钥

> **就这两条信息，没有第三步。**

### 第三步：双击 run.bat

配置完成后，双击 `run.bat`。第一次启动时，云端 AI 会自动：

1. 🔍 扫描你的电脑 — CPU、内存、操作系统、装了啥软件
2. 🛠️ 自动配置必要的工具
3. 🤝 告诉你接下来可以做什么

之后，你只需要在窗口中输入你想做的事：

```
您 > 帮我查看一下我的电脑配置
您 > 帮我在桌面上创建一个新项目
您 > 给我搭建一个聊天界面
```

## 用户故事

### 故事一：完全不懂技术的小明

小明下载了 Silent Protocol，双击 `setup.bat`，输入了 DeepSeek 的 API Key。

打开 `run.bat`，输入：

```
您 > 帮我检查一下系统
```

云端 AI 返回了 CPU、内存、磁盘信息。小明看懂了。

又输入：

```
您 > 给我装一个 Chatbox
```

云端 AI 自动下载安装 Chatbox，配置好本地 MCP Server 连接。五分钟后，小明有了一个本地的聊天界面。

**小明不需要知道什么是 MCP、什么是 Node.js、什么是 API 端点。他只需要说他想做什么。**

## 技术原理（非必需了解）

```
你的需求（自然语言）
    ↓
Cloud DeepSeek（云端 AI 大脑）
    ↓ 理解、分解、调度
Silent Protocol 网关（本地）
    ↓ 紧凑协议路由
本地工具链（CLI / MCP / 文件系统）
    ↓ 执行结果
返回给你（自然语言）
```

所有"麻烦的细节"都在网关层被消化了。你只负责：
1. 提出需求
2. 验收结果
3. 测试功能
4. 给出反馈

## 常见问题

### Q: 我不会编程，能用吗？
**能。** 这就是设计目标。你只需要会打字、会按回车。

### Q: 我需要自己配置 MCP Server 吗？
**不需要。** 云端 AI 会自动扫描你的电脑环境，自动配置一切。

### Q: 我需要懂什么是 API Key 吗？
**不需要。** 你知道从服务商那儿复制一串字符粘贴进去就行——就像注册任何网站一样。

### Q: 安全吗？
**安全。** 你的 API Key 只存在本地。云端 AI 只处理你明确要求的任务。

### Q: 如果出错了怎么办？
在 `run.bat` 窗口中告诉云端 AI 出错了，它会自动诊断和修复。你不需要自己查日志。
