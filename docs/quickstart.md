# 快速入门

> 本文档带你体验第一个完整的四方协作流程。
> 预计耗时：5 分钟。

---

## 场景：让你的 Claude Code 做一次文件统计

这是最简单的四方协作场景，用来验证整个链路是否畅通。

---

### 第一步：在 Chatbox 中说出你的需求

```
帮我统计 C:\tmmp\majiong-win 目录下一共有多少个文件。
```

---

### 第二步：观察背后发生了什么（你不需要做任何事）

内部自动发生的流程：

```
你 → Chatbox → API → 云端 DeepSeek
  ↓
云端 DeepSeek 理解需求 → 拆解为任务
  ↓
通过紧凑协议 → neca → 本地 Claude Code
  ↓
本地 Claude Code 执行: dir /s /a-d C:\tmmp\majiong-win | find /c ":"
  ↓
统计结果 → 通过紧凑协议 → 云端 DeepSeek
  ↓
云端 DeepSeek 组织成自然语言 → Chatbox → 你
```

---

### 第三步：看到结果

你会收到类似这样的回复：

```
C:\tmmp\majiong-win 目录下共有 42 个文件。
其中：
  - .py 文件: 12
  - .json 文件: 8
  - .txt 文件: 6
  - 其他: 16
```

---

### 尝菜与修正

如果你觉得格式不对，你只需要说：

```
按文件大小排序，从大到小。
```

内部协作再次自动运行，最终返回排序后的结果。

---

## 进阶场景：让你的 Claude Code 修改代码

```
帮我把 majiong-win 项目里所有的 print 语句改成 logging。
```

这个场景会触发更完整的四方协作：

1. **云端 DeepSeek** 拆任务：查找所有 print → 替换为 logging → 验证无语法错误
2. **本地 Claude Code** 执行：搜索文件、替换内容、运行测试
3. **云端 Claude API**（可选）：如果遇到难以处理的替换模式，被作为知识顾问调用

你只需要在最后尝菜时看一眼结果：

```
搞定了。修改了 5 个文件中的 23 处 print → logging。
测试通过，没有语法错误。你验证一下输出格式是否符合预期？
```

你说"日志格式再加个时间戳"——就又转起来了。

---

## 验证链路状态

随时可以通过以下方式确认各实体状态：

```
帮我检查一下整个链路的健康状态。
```

内部会调用 neca 的 ping 机制，依次检查：
- neca 网关：✅ 运行中
- 本地 Claude Code：✅ 空闲
- 云端 Claude API：✅ 可达（可选）
